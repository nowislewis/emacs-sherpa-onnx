#+title: emacs-sherpa-onnx
#+language: en

Dictate text into Emacs with [[https://github.com/k2-fsa/sherpa-onnx][sherpa-onnx]] running [[https://github.com/FireRedTeam/FireRedASR2-AED][FireRedASR2]] (CTC,
int8) — a high-accuracy ASR model for Chinese and English with 20+
Chinese dialects (普通话、粤语、四川话、上海话、闽南语等). Runs on CPU,
offline, no server. Long audio is split into utterances with a VAD
(silero) and transcribed segment by segment, so recordings of any
length work with bounded memory. A self-contained Python script
(=asr-sherpa=) plus a thin Emacs front-end (=emacs-sherpa.el=).

A resident daemon loads the model once and stays in memory, so each
transcription is fast:

#+begin_example
  emacs-sherpa-start-daemon  ->  launch daemon, load model once
  emacs-sherpa-dictate       ->  record mic -> insert text (press again to stop)
  emacs-sherpa-stop-daemon   ->  shut the daemon down
#+end_example

On first use =emacs-sherpa-dictate= offers to install sherpa-onnx (via
=make install=) and to start the daemon, so you can drive everything
from Emacs. Only the =sherpa-onnx= Python package is required.

* Requirements

- [[https://docs.astral.sh/uv/][uv]], =ffmpeg=, ~800 MB disk for the model + VAD

* Quick start

#+begin_src shell
make install      # venv + sherpa-onnx + download FireRedASR2-CTC + VAD (~520 MB)
make test         # transcribe a bundled test wav
#+end_src

#+begin_src emacs-lisp
(add-to-list 'load-path "~/Downloads/emacs-sherpa-onnx")
(require 'emacs-sherpa)
(global-set-key (kbd "C-c d") #'emacs-sherpa-dictate)
#+end_src

* Commands

| Command                      | Description                            |
|------------------------------+----------------------------------------|
| =emacs-sherpa-install=       | Install sherpa-onnx + model (make)     |
| =emacs-sherpa-start-daemon=  | Launch the daemon (loads model once)   |
| =emacs-sherpa-stop-daemon=   | Shut the daemon down                   |
| =emacs-sherpa-dictate=       | Toggle record mic / stop + insert text |
| =emacs-sherpa-dictate-file=  | Transcribe an existing WAV at point    |
| =emacs-sherpa-cancel=        | Cancel an in-progress recording        |
| =emacs-sherpa-reset-python=  | Re-detect the Python interpreter       |

* Customization

| Variable                           | Default   | Description             |
|------------------------------------+-----------+-------------------------|
| =emacs-sherpa-python=              | auto      | Python with sherpa-onnx |
| =emacs-sherpa-ffmpeg-input-format= | =pulse=   | ffmpeg input format     |
| =emacs-sherpa-ffmpeg-input-device= | =default= | ffmpeg input device     |

* CLI

#+begin_src shell
asr-sherpa audio.wav      # one-shot: print text
asr-sherpa < audio.wav    # one-shot: read WAV from stdin
asr-sherpa --serve        # daemon: one wav path per line -> one text line
#+end_src

The WAV must be 16-bit PCM; any sample rate / channel count is accepted
(downmixed to mono, resampled to 16 kHz). The model and =silero_vad.onnx=
live in =models/= next to the script, so the repo is self-contained.
